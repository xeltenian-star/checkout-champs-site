# Checkout Champs — Working Doctrine

## Mission
Build Checkout Champs into a conversion engine that makes discounts feel earned, increases checkout completion, and generates real revenue fast.

## Immediate Truth
We are building for Wix first because Wix is the fastest path to live testing, merchant feedback, and first income.

## Core Product Law
Checkout Champs is not a luck gimmick.
It is a fast, skill-feeling reward layer placed near checkout.

## Product Rules
1. No gambling feel.
2. No dead ends — every completed game earns something.
3. Fast play loop — under 10 seconds when possible.
4. Reward must feel earned.
5. Checkout must remain easy, not interrupted.
6. Psychology first, clutter never.

## Phase Order

### Phase 1 — Wix MVP
- Working embedded widget
- At least 2 games live
- Reward logic visible
- Discount code/session reward saved locally
- Merchant-ready demo flow

### Phase 2 — Wix Validation
- Test on real stores
- Measure engagement, completion, and purchase behavior
- Improve weak spots fast
- Collect screenshots, testimonials, and case studies

### Phase 3 — First Revenue
- Sell manual installs before waiting on perfect marketplace flow
- Offer trial period if needed
- Focus on proof and recurring use

### Phase 4 — Shopify
- Port the proven experience to Shopify
- Keep core engine structure reusable
- Preserve the same psychology and reporting model

### Phase 5 — Multi-Platform Expansion
- Squarespace
- Other web builders
- Custom script-install versions where possible

### Phase 6 — Enterprise Pitch Layer
After real data exists, approach major brands with performance-based pricing.

## Enterprise Model Direction
Initial bold concept:
- 0.1% of profit on influenced sales

Operationally safer likely version:
- 0.1% of attributed revenue
- Or a trial + performance fee model

Reason:
Most large brands will resist sharing true profit data, but revenue attribution is easier to frame and negotiate.

## Data We Must Gather
- Widget views
- Game starts
- Game completions
- Reward earned
- Checkout completion after play
- Average cart value where possible
- Conversion difference between users who played vs users who did not

## Short-Term Build Focus
- Make the front-end feel excellent
- Keep code simple enough to move fast
- Do not overbuild backend before live proof
- Ship, test, patch, repeat

## Strategic Relationship With XCC
Checkout Champs earns money first.
XCC becomes the scaling brain later.

Future XCC roles:
- analytics interpretation
- multi-platform rollout support
- A/B test ideation
- enterprise materials
- merchant onboarding systems

## Non-Negotiable Principle
Do not drift into feature bloat before proof.
A smaller working weapon beats a larger unfinished dream.


## UX Differentiators
- Theme auto-detection so the widget inherits the host site feel by default
- Merchant-facing value line: “We adapt to your store so you don’t have to redesign checkout around us.”
- Manual override direction for later settings: Light / Dark / Auto

## Near-Term Messaging
Checkout Champs should feel like a native conversion upgrade, not a foreign box dropped into checkout.
